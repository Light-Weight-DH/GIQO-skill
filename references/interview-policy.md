# Interview and Assumption Policy

GIQO should reduce user burden.

## Ask when

- The decision is owner-level.
- A wrong default would cause rework.
- The source material contains conflicting instructions.
- The answer changes architecture, data model, platform, or scope.

## Do not ask when

- The codebase or references answer it.
- A standard default is reversible.
- The detail belongs in implementation rather than planning.
- The user explicitly says to proceed without questions.

## Assumption format

```markdown
## A-001: <short name>

- Assumption: ...
- Reason: ...
- Impact if wrong: ...
- Revision path: ...
```
