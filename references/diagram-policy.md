# Mermaid Diagram Policy

Use Mermaid as an implementation aid.

## ERD

Use when the system has persistent entities or relationships.

## Flowchart

Use when users, business logic, or system state branches.

## Sequence diagram

Use when two or more actors/services exchange messages.

## Gantt

Use when schedule, milestones, or dependency timing matters.

## Rules

- Every diagram needs a short explanation.
- Diagrams must match the text.
- Do not create decorative diagrams.
- If Mermaid syntax is uncertain, prefer simpler valid syntax.
