# Output Quality Bar

GIQO output is complete when an implementation agent can start without another interview.

## Required qualities

- Source-backed: every major decision traces to input or an assumption.
- Selective: unnecessary documents are omitted.
- Navigable: `00_INDEX.md` explains reading order.
- Actionable: `05_IMPLEMENTATION_PLAN.md` has ordered work units.
- Explicit: non-goals and unresolved decisions are visible.
- Reviewable: UI work has states, accessibility notes, and comments when available.

## Must not include

- Generic filler sections.
- Mermaid diagrams that repeat obvious text.
- Implementation tasks without acceptance criteria.
- Hidden assumptions.
- Platform-specific instructions unless the target platform is known.
